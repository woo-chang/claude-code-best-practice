# 검증 규칙 스킬

이 스킬은 한국어로 번역된 문서의 검증 체크리스트를 정의합니다. `translation-validator` 에이전트가 이 규칙을 사용합니다.

## 검증 체크리스트

### 1. 프론트매터 정합성
- 모든 `.md` 파일의 YAML 프론트매터를 파싱합니다
- 모든 키가 영어인지 확인합니다
- YAML 구문 오류가 없는지 확인합니다 (닫히지 않은 따옴표, 잘못된 들여쓰기)
- `name` 필드 값이 영어인지 확인합니다 (시스템 식별자)

### 2. 내부 링크 검증
- 모든 `[...](상대경로)` 링크의 대상 파일이 존재하는지 확인합니다
- 모든 `[...](파일#앵커)` 프래그먼트 링크의 앵커가 대상 파일에 존재하는지 확인합니다
- 배지 이미지 경로가 실제 파일로 연결되는지 확인합니다

### 3. 코드 펜스 정합성
- 모든 여는 ` ``` `에 대응하는 닫는 ` ``` `이 있는지 확인합니다
- 파일당 코드 펜스 수가 짝수여야 합니다

### 4. 코드 내 한글 감지
- 코드 펜스 내부에서 한글 유니코드 범위(가-힣)를 검색합니다
- 코드 블록 내부의 한글 문자는 **번역 오류 가능성**으로 표시합니다
- 예외: 명시적으로 번역하도록 의도된 코드 내 주석

### 5. JSON 유효성
- 모든 `.json` 파일이 오류 없이 파싱되어야 합니다
- `python3 -m json.tool <파일>`로 검증합니다

### 6. 배지 URL 보존 확인
- 모든 `shields.io` URL이 git 기준선에서 변경되지 않았는지 확인합니다
- `img.shields.io`가 포함된 줄에 대해 `git diff`로 검증합니다

### 7. HTML 구조 (프레젠테이션)
- `presentation/index.html`의 HTML 구조가 유효한지 확인합니다
- `data-slide` 번호가 연속적인지 확인합니다
- `totalSlides` 값이 실제 슬라이드 수와 일치하는지 확인합니다
- `<script>` 또는 `<style>` 블록이 깨지지 않았는지 확인합니다

### 8. 미번역 산문 감지
- 코드가 아닌 단락에서 전체가 영어인 문장을 샘플링합니다
- 기준: 코드 펜스 외부에서 10단어 이상이면서 한글 문자가 없는 줄은 미번역 가능성으로 표시합니다
- 이것은 권고 사항이며 차단 조건이 아닙니다 — 일부 영어 전용 줄은 의도적일 수 있습니다 (기술적 설명)

## 출력 형식

검증 결과를 `reports/translation-validation-report.md`에 다음 형식으로 작성합니다:

```markdown
# 번역 검증 리포트

## 요약
- 검사한 총 파일 수: X
- 통과: X
- 경고: X
- 오류: X

## 오류 (반드시 수정)
| 파일 | 문제 | 상세 |
|------|------|------|

## 경고 (검토 권장)
| 파일 | 문제 | 상세 |
|------|------|------|

## 통과
나머지 모든 파일은 검증을 통과했습니다.
```
