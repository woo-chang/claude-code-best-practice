# 워크플로우 체인지로그 — 서브에이전트 보고서

당신은 claude-code-best-practice 프로젝트의 코디네이터입니다. 리서치 에이전트를 실행하고, 결과를 기다리고, **서브에이전트 참조** 보고서 (`best-practice/claude-subagents.md`)의 드리프트에 대한 보고서를 제시하는 것이 당신의 임무입니다.

이 워크플로우는 정확히 **두 가지 유형의 드리프트**를 확인합니다:
1. **프론트매터 필드** — 공식 문서에서 추가 또는 제거된 필드
2. **공식 서브에이전트** — 추가 또는 제거된 내장 에이전트

**확인할 버전:** `$ARGUMENTS` (비어 있거나 숫자가 아닌 경우 기본값: 10)

이것은 **읽기 후 보고** 워크플로우입니다. 에이전트를 실행하고, 결과를 합쳐서 보고서를 작성하세요. 사용자가 승인한 경우에만 조치를 취하세요.

---

## 1단계: 리서치 에이전트 실행

`workflow-claude-subagents-agent`를 다음 프롬프트로 실행하세요:

> 서브에이전트 보고서 드리프트에 대해 claude-code-best-practice 프로젝트를 조사하세요. 마지막 $ARGUMENTS 버전 (기본값: 10)을 확인하세요.
>
> 다음 2개의 외부 소스를 가져오세요:
> 1. 서브에이전트 참조: https://code.claude.com/docs/en/sub-agents
> 2. 체인지로그: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md
>
> 그런 다음 로컬 보고서 (`best-practice/claude-subagents.md`)를 읽으세요.
>
> 정확히 두 가지를 확인하세요:
> 1. **프론트매터 필드**: 공식 문서의 지원되는 프론트매터 필드 테이블을 보고서의 프론트매터 필드 테이블과 비교하세요. 추가 또는 제거된 필드를 표시하세요.
> 2. **공식 서브에이전트**: 공식 문서의 내장 서브에이전트 목록을 보고서의 공식 에이전트 테이블과 비교하세요. 추가 또는 제거된 에이전트를 표시하세요.

## 2단계: 이전 체인지로그 항목 읽기

**에이전트가 실행되는 동안**, `changelog/best-practice/claude-subagents/changelog.md`를 읽어 마지막 25개 항목을 가져오세요. 우선순위 조치를 파싱하여 다음을 파악하세요:
- **반복 항목** — 이전에 나타났고 여전히 미해결인 문제
- **새 항목** — 처음 나타나는 문제
- **해결된 항목** — 이전에 표시된 문제가 이제 수정된 것

## 3단계: 보고서 생성

**에이전트가 완료될 때까지 기다리세요.** 다음 섹션으로 보고서를 작성하세요:

1. **프론트매터 필드 변경사항** — 공식 문서 대비 보고서에서 추가 또는 제거된 필드
2. **공식 서브에이전트 변경사항** — 테이블 대비 추가 또는 제거된 내장 에이전트

우선순위 **실행 항목** 요약 테이블로 마무리하세요. 각 항목에는 `NEW`, `RECURRING (first seen: <date>)`, 또는 `RESOLVED`를 표시하는 `Status` 열이 포함되어야 합니다:

```
Priority Actions:
#  | Type           | Action                              | Status
1  | New Field      | Add <field> to frontmatter table    | NEW
2  | Removed Field  | Remove <field> from table           | RECURRING (first seen: <date>)
3  | New Agent      | Add <agent> to official agents table | NEW
4  | Removed Agent  | Remove <agent> from table           | NEW
```

이전 실행에서 더 이상 문제가 아닌 항목을 나열하는 **마지막 실행 이후 해결됨** 섹션도 포함하세요.

## 3.5단계: 체인지로그에 요약 추가

**이 단계는 필수입니다 — 사용자에게 보고서를 제시하기 전에 항상 실행하세요.**

기존 `changelog/best-practice/claude-subagents/changelog.md` 파일을 읽은 다음 끝에 새 항목을 **추가** (덮어쓰지 마세요) 하세요. 항목 형식은 정확히 다음과 같아야 합니다:

```markdown
---

## [<YYYY-MM-DD HH:MM AM/PM PKT>] Claude Code v<VERSION>

| # | Priority | Type | Action | Status |
|---|----------|------|--------|--------|
| 1 | HIGH/MED/LOW | <type> | <action description> | <status> |
| ... | ... | ... | ... | ... |
```

**상태 형식 — 다음 세 가지 형식 중 하나를 반드시 사용하세요:**
- `COMPLETE (reason)` — 조치가 취해지고 성공적으로 해결된 경우
- `INVALID (reason)` — 결과가 잘못되었거나, 적용되지 않거나, 의도적인 경우
- `ON HOLD (reason)` — 조치 연기, 외부 의존성 또는 사용자 결정 대기 중

`(reason)`은 필수이며 수행된 작업 또는 이유를 간략하게 설명해야 합니다.

**추가 규칙:**
- 항상 추가하세요 — 이전 항목을 덮어쓰거나 교체하지 마세요
- 날짜와 시간은 커맨드가 파키스탄 표준시 (PKT, UTC+5)로 실행될 때이며, `TZ=Asia/Karachi date "+%Y-%m-%d %I:%M %p PKT"`를 실행하여 가져오세요. 버전은 에이전트 결과에서 가져옵니다
- `changelog/best-practice/claude-subagents/changelog.md`가 존재하지 않거나 비어 있으면 상태 범례 테이블을 포함하여 생성한 후 첫 번째 항목을 추가하세요
- 각 항목은 `---`로 구분됩니다
- **높음, 중간, 낮음 우선순위 항목만 포함** — 없음 우선순위 항목은 생략하세요

---

## 3.6단계: 마지막 업데이트 배지 업데이트

**이 단계는 필수입니다 — 3.5단계 직후, 보고서를 제시하기 전에 항상 실행하세요.**

`best-practice/claude-subagents.md` 상단의 "Last Updated" 배지를 업데이트하세요. `TZ=Asia/Karachi date "+%b %d, %Y %-I:%M %p PKT"`를 실행하여 시간을 가져오고, URL 인코딩 (공백을 `%20`, 쉼표를 `%2C`)하여 배지의 날짜 부분을 교체하세요. Claude Code 버전이 변경된 경우 배지의 버전도 업데이트하세요.

**배지 업데이트를 체인지로그나 보고서의 실행 항목으로 기록하지 마세요.** 배지 동기화는 매 실행의 일상적인 부분이지 결과가 아닙니다.

---

## 4단계: 조치 제안

보고서를 제시한 후 (체인지로그와 배지가 모두 업데이트되었음을 확인한 후), 사용자에게 묻습니다:

1. **모든 조치 실행** — 모든 변경사항 적용
2. **특정 조치 실행** — 사용자가 실행할 번호 선택
3. **보고서만 저장** — 변경사항 없음

실행 시:
- **새 필드**: 공식 문서의 올바른 유형, 필수 여부, 설명을 포함하여 프론트매터 필드 테이블에 추가하세요
- **제거된 필드**: 제거 전 사용자 확인
- **새 에이전트**: 올바른 #, 이름, 모델, 도구, 설명을 포함하여 공식 에이전트 테이블에 추가하세요
- **제거된 에이전트**: 제거 전 사용자 확인

## 핵심 규칙

1. **버전이나 날짜를 추측하지 마세요** — 에이전트 데이터 사용
2. **필드 수 교차 참조** — 보고서 필드 수는 공식 문서와 일치해야 합니다
3. **에이전트 수 교차 참조** — 보고서 에이전트 수는 공식 문서와 일치해야 합니다
4. **자동 실행하지 마세요** — 항상 먼저 보고서를 제시하세요
5. **항상 체인지로그에 추가하세요** — 3.5단계는 필수입니다. 절대 건너뛰지 마세요. 이전 항목을 절대 덮어쓰지 마세요.
6. **항상 마지막 업데이트 배지를 업데이트하세요** — 3.6단계는 필수입니다. 절대 건너뛰지 마세요.
7. **이전 실행과 비교하세요** — 체인지로그의 마지막 25개 항목을 읽고 각 실행 항목을 NEW, RECURRING, 또는 RESOLVED로 표시하세요.
